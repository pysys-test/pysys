 52          """The parent class of all PySys system tests. 
 53   
 54          BaseTest is the parent class of all PySys system tests, and should be subclassed to provide  
 55          an implementation of the execute() method. The class is instantiated with information on the  
 56          test input, reference and output data locations, the user defined mode the test is to be executed  
 57          in, and the user defined extra arguments required for test execution.  
 58   
 59          The class provides important utility functions for process management, test timing, test validation  
 60          and storage of test validation steps. Validation of the test can be performed multiple times through  
 61          the assert* functions, building up an internal list of individual validation outcomes. Currently  
 62          supported test outcomes are; 
 63           
 64            SKIPPED:     An execution/validation step of the test was skipped (e.g. deliberately) 
 65            BLOCKED:     An execution/validation step of the test could not be run (e.g. a missing resource) 
 66            DUMPEDCORE:  A process started by the test produced a core file (unix only) 
 67            TIMEDOUT:    An execution/validation step of the test timed out (e.g. process deadlock) 
 68            FAILED:      A validation step of the test failed 
 69            NOTVERIFIED: No validation steps were performed 
 70            INSPECT:     A validation step of the test requires manual inspection 
 71            PASSED:      A validation step of the test passed 
 72           
 73          The overall outcome of the test is based on a precedence of the outcomes in the list, based on the order 
 74          as given above i.e. if three validation steps were performed in the test with outcomes TIMEDOUT, FAILED,  
 75          PASSED the overall outcome would be TIMEDOUT; a test only passes when all validation steps pass.  
 76            
 77          """
